Help & User Guide — How to Use This Tool

IMO DCS & CII Verification Tool — Complete Guide
This guide explains every tab, what each column means, how values are calculated, and important cautions to avoid errors when preparing your IMO DCS report and CII rating under MARPOL Annex VI.

🚢 1. Ship Particulars

Establishes the vessel's identity, design parameters and reporting period. These values drive the CII reference line (the "a, c, d" parameters) and the EEXI/EEDI benchmarks used on the CII Results tab, so they must be entered accurately before any other tab.

Field / ColumnDescriptionCaution ⚠
Vessel NameFull vessel name as registered; used to label the exported Annual Report and XML file.Must match the vessel's certificates exactly for DCS submission.
IMO Number7-digit IMO unique vessel identifier used for DCS submission matching and report labelling.IMO number must be exactly 7 digits — a wrong number will cause the submission to be rejected.
Ship TypeSelects the CII reference parameters (a, c, d1–d4) per MEPC.354(78). Also controls which correction factor sections are shown (e.g. tanker-only sections).Wrong ship type silently changes the entire CII rating — verify against the vessel's IOPP/IEE certificate.
DWT (MT)Deadweight tonnage used as the capacity denominator for DWT-based ship types (bulk carriers, tankers, gas carriers, container ships, general cargo, LNG carriers, etc.).Use the summer load line DWT from the International Load Line Certificate, not commercial deadweight.
Gross TonnageUsed as capacity denominator for GT-based ship types (Ro-ro, cruise passenger ships). Also flows to the Annual Report header.Use the value on the International Tonnage Certificate — not the vessel's commercial GT.
Flag StateThe flag administration under which the vessel is registered; appears on the Annual Report for submission routing.Ensure this matches the vessel's current flag — a flag change mid-year may create a partial-period reporting obligation.
Ice ClassTriggers ice-class correction factor eligibility for the CII voyage adjustment (FCvoyage). Options: IA Super, IA, IB, IC.Only apply if the class notation is current and valid for the full reporting year — expired notations do not qualify.
CompanyShipowner or technical/commercial manager name as declared in the DCS submission; appears on the Annual Report.Use the name exactly as listed on the Document of Compliance (DOC).
Year of DeliveryThe calendar year the vessel was delivered from the shipyard; used to determine EEDI/EEXI applicability thresholds.Verify against the vessel's Builder's Certificate — affects whether an EEDI or EEXI value is required.
Net TonnageIncluded in the Annual Report header for completeness; not used directly in CII or DCS calculations.Take from the International Tonnage Certificate.
Attained EEDIEnergy Efficiency Design Index value from the EEDI Technical File; shown for reference on the Annual Report.Required for vessels delivered on or after 1 Jan 2013. Leave blank only if the ship is exempt.
Attained EEXIEnergy Efficiency Existing Ship Index value from the EEXI Technical File; shown for reference on the Annual Report.Required for ships ≥400GT from 1 Jan 2023. An EEXI correction section is also available under CII Correction Factors.
Main Engine(s) Rated Power (kW)Maximum continuous rating (MCR) of the main engine(s); used for EEXI correction calculations in the Correction Factors tab.Use the rated power from the Engine International Air Pollution Prevention (EIAPP) Certificate, not a derated value.
Auxiliary Engine(s) Rated Power (kW)Aggregate rated power of the auxiliary/generator engines; used for EEXI correction calculations.Include all generators that contribute to DCS fuel consumption; exclude emergency generators.
Reporting Year (DCS)Calendar year of data collection (2023–2030). Sets the CII reduction factor and the Required CII reference line used throughout the tool.The reduction factor is year-specific and increases annually to 2030 — always verify this is set correctly before running calculations.
Start / End Date DCSThe official DCS data collection period — normally 1 Jan to 31 Dec of the reporting year.For full-year reporting, leave as 1 Jan–31 Dec. Do not manually shorten these dates for partial-year periods — use the Partial Period toggle instead.
Start / End Date Annual CIIThe CII assessment period — normally identical to the DCS period but may differ for partial periods (e.g. change of flag, delivery).Partial-year periods (delivery, sale, flag change) must use the "Partial Period" toggle so the Required CII reference line is correctly scaled.
Reporting Period — Full Year / Partial PeriodToggle between a standard full-year period and a custom partial period. When Partial Period is selected, enter start and end dates; the tool scales the Required CII reference line proportionally.Do not use the Partial Period toggle just to adjust dates — only use it when the vessel genuinely operated for less than the full calendar year under this flag/owner.
CII Reference Line PreviewDisplays the Required CII, A/B, B/C, C/D and D/E boundary values live as Ship Type, DWT/GT and Reporting Year are entered — no calculation button needed.If this panel shows an error, the CII Results tab will also be incorrect — resolve missing inputs first.
Innovative Technology (MEPC.1/Circ.896)Free-text field for noting any innovative energy-saving technologies installed on the vessel (e.g. wind-assisted propulsion, air lubrication). Printed on the Annual Report.Enter "N/A" if not applicable — leave blank only if the field is genuinely unknown.
CII for Trial PurposeRecords any voluntary CII trial metrics being monitored on the vessel during the reporting year.Optional — enter "None" if not applicable.
EEPI / cbDIST / clDIST / EEOIOptional operational efficiency metrics (Energy Efficiency Performance Indicator, berth-distance and length-distance indices, Energy Efficiency Operational Indicator) that appear on the Annual Report when provided.These are voluntary metrics — leave blank if not being tracked. Units are labelled next to each field.
📋 2. Voyage Data

The core logbook of voyages and per-voyage fuel consumption by consumer (Main Engine, Aux Engine, Inert Gas Generator, Fired Boiler, Others). Distance, hours and laden condition here feed directly into total fuel consumed, distance travelled and the CII transport work denominator. Each row represents one voyage leg.

ColumnDescriptionCaution ⚠
# (Row number)Auto-assigned sequential row index. Used for reference only — does not affect calculations.Rows are always processed in entry order; reordering is not supported.
IMO No.Vessel IMO number auto-carried from Ship Particulars. Populated per row for DCS report completeness.If blank here, ensure IMO Number is entered in Ship Particulars first.
Voyage No.Sequential voyage or leg reference number as per the vessel's voyage records or DCS logbook.Use a consistent numbering convention throughout the reporting year — auditors may cross-check against voyage reports.
From Country / From PortDeparture country and port name for this leg. Used for EU MRV classification and port-state reporting.Required for EU MRV legs — leaving blank may under-report EU voyage emissions.
EU? Dep (EU Departure flag)Mark Yes if the departure port is within an EU/EEA member state. Determines whether this leg's emissions count toward EU MRV reporting.Apply the flag at the port level per EU MRV Regulation (EU) 2015/757 — not at country-border crossing level.
To Country / To PortArrival country and port name for this leg.Required for EU MRV; must reflect the actual port of call, not a waypoint.
EU? Arr (EU Arrival flag)Mark Yes if the arrival port is within an EU/EEA member state. Together with EU Dep, determines MRV leg classification (between EU, departing EU, arriving EU, non-EU).A leg with both EU Dep and EU Arr is classified "between EU" and counts 100% toward EU MRV. One flag only = 50%.
Start EventThe nautical event marking the start of this leg (e.g. "COSP" — Commencement of Sea Passage, "Departure", "All Fast", "Anchor Up").Event definitions must be applied consistently across all rows — inconsistent definitions break the hours calculation and may cause audit queries.
Start DateDate (and time where applicable) of the start event for this voyage leg. Drives the hours calculation together with End Date.Ensure dates are in chronological order — overlapping or reversed date ranges are flagged by the Verification tab.
End EventThe nautical event marking the end of this leg (e.g. "EOSP" — End of Sea Passage, "All Fast", "Anchor Down", "Pilot On Board").The Start Event of the next row should logically follow the End Event of this row — gaps or overlaps will flag in Verification.
End DateDate (and time where applicable) of the end event for this voyage leg.End Date must be on or after Start Date. If a voyage crosses midnight, the date must advance accordingly.
Dist (nm) — DistanceObserved nautical miles logged for this leg. Summed across all rows to give Total Distance Travelled, the CII denominator.Use logged/observed distance (from noon reports or GPS track), not the great-circle or charter-party distance. Discrepancies flag in Verification.
HoursHours underway for this leg. Used for average speed plausibility checks in Verification and for port/sea time split reporting.Must be consistent with Start/End dates. Hours entered in minutes (×60 error) is the most common data entry mistake.
Under way / Not Under wayIndicates whether the vessel was propelling (sea passage) or stationary (port, anchor, drifting). Determines how distance and fuel are categorised in DCS reporting.Fuel consumed while "Not Under way" is still counted in the CII CO₂ numerator — only voyage adjustment corrections can exclude it.
ActivityOperational activity type for this leg (e.g. Laden, Ballast, Port, Anchor, Drifting, STS). Feeds the EU MRV activity split and supports correction factor eligibility.Mislabelling ballast voyages as laden inflates transport work and artificially lowers (improves) the attained CII.
Cargo (MT)Cargo mass in metric tonnes carried on this leg. Used for transport-work-based CII variants and EEOI calculations where applicable.Only relevant for ship types using cargo-based CII metrics — enter 0 if not applicable for your ship type and formula.
Fuel Type Columns — ME (Main Engine)Fuel mass (metric tonnes) consumed by the Main Engine on this leg, per fuel type column. The main propulsion consumer; typically the largest fuel entry.Do not include fuel transferred to service tanks but not yet burned — record only actual consumption.
Fuel Type Columns — AE/Gen (Auxiliary Engines / Generators)Fuel consumed by auxiliary engines and generators providing electrical power on this leg.Separate ME and AE consumption must be tracked per fuel type if different fuels are used in each — mixing them under one consumer will distort the CO₂ split.
Fuel Type Columns — IGG (Inert Gas Generator)Fuel consumed by the inert gas generator, where applicable (typically tankers and gas carriers).Leave as 0 for ship types without an IGG system — entering values here for non-eligible vessels will inflate total CO₂.
Fuel Type Columns — F.Boiler (Fired Boiler)Fuel consumed by oil-fired boilers for steam generation (heating, cargo conditioning, etc.).If a boiler correction factor (FCboiler) is being claimed, the fuel entered here must match the correction factor input — double-check both tabs.
Fuel Type Columns — Others (customisable)Catch-all consumer column for any fuel consumption not captured by ME, AE, IGG or Boiler (e.g. incinerators, deck machinery). The column header can be renamed by clicking it.Only use this column for fuel that is definitively consumed by a consumer not already listed — mixing categories creates audit discrepancies.
⚡ Total (per fuel type)Auto-calculated sum of ME + AE + IGG + Boiler + Others for that fuel type on this leg. Read-only.If this total looks unexpectedly high or low, check each consumer column individually rather than editing this cell.
Total Fuel (MT)Auto-calculated grand total fuel consumed across all fuel types on this leg. Read-only. Flows into the Fuel Consumption tab totals.Cross-check against the vessel's bunker survey / ROB (Remaining on Board) records for this leg.
Del (Delete)Removes this voyage row from the table. Cannot be undone — export XML first if you need a backup.Deleting a row permanently removes its fuel data from all downstream calculations.
Toolbar tip: + Add Row adds a blank leg · Sample Data loads demo rows for testing · Clear wipes the table · Template downloads a 1000-row Excel template · Import reads a completed Excel template back into the table · ⚡ Calculate & View Results runs all calculations and advances to Correction Factors.
🔧 3. CII Correction Factors

Optional adjustments permitted under MEPC.355(78) Annex 1 Appendix to subtract fuel consumption attributable to non-propulsion purposes from the CII numerator. The corrected attained CII = Σ(CFj × FCj) / (Capacity × Distance). Applicable mainly to tankers, gas carriers and reefer-capable ships.

Section / FieldDescriptionCaution ⚠
Voyage Adjustment (FCvoyage)Deducts fuel and distance for legs flagged as voyage adjustments (MARPOL Reg. 3.1 exemptions, ice navigation). Enabled by marking rows in Voyage Data with the "Voyage Adj?" flag.Must be approved by the Administration or RO before claiming — unsupported claims are a primary cause of Statement of Compliance refusal.
Transferred Fuel — STS (TF STS)Correction for tankers engaged in ship-to-ship (STS) operations; deducts fuel consumed during STS cargo transfer, voyage to/from STS location and associated waiting periods.Only applies if the vessel meets the STS definition in MEPC.355(78) section 2.8 — confirm with the RO before applying.
Transferred Fuel — Shuttle Tanker (TF Shuttle)Correction for shuttle tankers equipped with dynamic positioning (DP) and dedicated cargo handling equipment — deducts fuel for the DP loading operation portion of each voyage.Cannot be used simultaneously with TF STS for the same vessel. Only applies when the shuttle tanker definition in MEPC.355(78) section 2.9 is met.
FCelectrical — Reefer Container CorrectionDeducts fuel energy attributed to powering refrigerated containers, either via directly monitored kWh consumption or approximated using the SFOC method (kWh × SFOC / 1,000,000). Sea-day and port-day inputs are available for the approximation method.Only valid if reefer plugs are separately metered or the approved MEPC.355(78) Appendix 1 Part A approximation formula is used and documented.
FCelectrical — Cargo Cooling / ReliquefactionDeducts fuel for cargo cooling or reliquefaction systems on gas carriers and LNG carriers (electrical consumption method, kWh × SFOC).Gas and LNG carriers only — section auto-hides for non-eligible ship types. Approximate SFOC can be used if direct monitoring is unavailable.
FCelectrical — Cargo Discharge PumpsDeducts fuel for electrically-driven cargo discharge pumps on tankers (kWh × SFOC).Applicable only to tankers where discharge pumps draw power from the main/auxiliary engines included in DCS fuel accounting.
FCboiler (Boiler Correction)Deducts fuel consumed by oil-fired boilers where the steam output is used for cargo heating or other non-propulsion purposes. Enter boiler fuel consumption by fuel type.Requires metered boiler fuel consumption records (flow meters or dedicated tank). Auditors will request evidence — estimated values are not accepted.
FCothers (Others Correction)Deducts fuel for other non-propulsion consumers not covered by the above categories, where an IMO-approved correction method applies.Requires a specific regulatory reference (e.g. MEPC circular) justifying the deduction — do not use as a general adjustment.
fi (Ice Class Factor)Scalar multiplier applied to the CII formula for ice-classed vessels to account for the structural weight penalty from ice-strengthened hull. Set automatically based on the Ice Class entered in Ship Particulars.Only modify if a specific IMO circular overrides the default value for your ice class notation.
fm (Capacity Factor)Correction factor for vessels that cannot utilise their full rated capacity due to structural or regulatory constraints (e.g. DWT limited by freeboard).Default is 1.0 (no correction). Consult the RO before applying a non-unity value — supporting documentation is required.
fc (Cubic Capacity Factor)Applicable to gas carriers where the capacity measure is based on cubic metres rather than DWT — adjusts the CII denominator accordingly.Gas carriers only. Verify the applicable fc value from the vessel's IEE Certificate or class certificate.
fivse (Innovative Energy Saving Technology Factor)Correction factor for vessels fitted with an approved innovative energy-saving technology (e.g. wind-assisted propulsion, waste heat recovery) as recognised under MEPC.1/Circ.896.Requires prior approval from the Administration — the technology must be listed in Ship Particulars (Innovative Technology field) and supported by an approved SEEMP Part II.
EEXI Correction SectionAdjustment for vessels that have implemented an Engine Power Limitation (EPL) or Shaft Power Limitation (ShaPoLi) to comply with EEXI — corrects the CII calculation to reflect the effective power limitation.Only applicable if the vessel has a verified EEXI with an EPL/ShaPoLi installed and certified. Requires rated power inputs from Ship Particulars.
Corrected CII ResultDisplays the final corrected attained CII after all applicable correction factors have been applied, along with the correction breakdown table and updated rating band (A–E).If the corrected CII differs unexpectedly from the standard CII, verify that no correction factors were applied in error.
⛽ 4. Fuel Consumption

Aggregates all voyage-level fuel entries into total mass consumed per fuel type and per consumer for the reporting period — the basis for both the DCS fuel report and the CO₂ emissions calculation (Fuel Consumed × Cf). All values are auto-calculated from Voyage Data; no manual entry is required here.

Column / ElementDescriptionCaution ⚠
Fuel Type (HFO / MGO / LFO / LNG / Custom…)Each fuel column is assigned a fuel type and emission factor (Cf) in the Voyage Data tab header. The assignment determines the CO₂ factor applied to all consumption entered in that column.Custom fuels need an IMO-recognised CO₂ emission factor (per MEPC.1/Circ.795 or equivalent) — do not use estimated values.
Consumer breakdown (ME / AE / IGG / F.Boiler / Others)Total fuel consumed by each consumer type for the full reporting period, per fuel type. Summed from all voyage rows.If a consumer total looks unexpectedly zero or very high, return to Voyage Data and check individual row entries for that consumer column.
Total Fuel Consumed (t)Grand total fuel mass for each fuel type across all consumers and all voyage legs. This is the primary DCS fuel figure.Must reconcile with bunker delivery notes (BDN) and tank sounding records — a discrepancy here will fail the Verification checks.
CO₂ Equivalent (tCO₂)Total CO₂ = Total Fuel Consumed (t) × Emission Factor Cf for that fuel type. Summed across all fuel types to give Total CO₂ Emitted.Changing a fuel type assignment after data entry immediately recalculates all CO₂ — re-check totals and re-run calculations if you change any fuel assignment.
% ShareEach fuel type's contribution to total CO₂ emissions as a percentage. Useful for identifying the dominant fuel driving the CII result.If the % share distribution looks unexpected, verify that the correct Cf emission factor is assigned to each fuel type.
EU MRV ClassificationSplits total CO₂ into EU MRV categories: voyages between EU ports (100%), departing EU (50%), arriving EU (50%), non-EU voyages (0%) and port emissions — based on the EU flags entered in Voyage Data.If the EU MRV total appears too low, check that EU Dep and EU Arr flags are correctly set for all applicable voyage rows.
Visual Analytics ChartsAuto-generated charts showing fuel consumption by consumer type, EU MRV split, activity (underway vs not underway) and fuel type mix (⛽ Fuel Type Mix). Chart legends show the actual fuel type names assigned in Voyage Data.Charts are for visualisation only — they do not affect calculations. If legend names appear as codes rather than fuel names, re-run the calculation from Voyage Data.
📊 5. CII Results

Calculates Attained CII (AER, gCO₂/dwt-nm), Required CII (reference line × reduction factor for the year), and the resulting rating band A–E as per MEPC.354(78) and MEPC.355(78).

Metric / ElementDescriptionCaution ⚠
Total CO₂ EmittedSum of CO₂ from all fuels after correction factor deductions.Verify correction factors were applied correctly — over-deduction is a common audit finding.
Total Distance TravelledSum of voyage distances for the reporting period.Distance run at anchor/drifting should generally be excluded per IMO guidance.
Attained CII (AER)CO₂ (g) ÷ (Capacity × Distance) — the ship's actual annual efficiency ratio.Lower is better; small distance/capacity errors disproportionately swing this value.
Required CIIReference line value reduced by the year's required reduction factor (e.g. 11% in 2026, rising annually to 2030).The reduction factor is year-specific — confirm the correct Reporting Year is set on Ship Particulars.
CII Rating Band (A–E)A=major superior, B=minor superior, C=moderate, D=minor inferior (3 consecutive years triggers SEEMP corrective action plan), E=inferior (immediate corrective action plan required).D for 3 consecutive years or any single E rating creates a mandatory SEEMP Part III corrective action obligation.

✅ 6. Verification

Automated sanity checks that flag data inconsistencies before submission to the Administration / RO (Recognised Organisation), reducing the risk of a Statement of Compliance being withheld.

CheckDescriptionCaution ⚠
Date sequence checksConfirms voyage start/end dates are chronological and non-overlapping.Overlapping voyages usually indicate a copy-paste error in the Voyage Data tab.
Fuel balance checksCross-checks total fuel consumed against expected consumption per consumer.Large unexplained variances should be resolved before export, not after.
Missing field checksHighlights required fields left blank across all tabs.Fields marked with a red asterisk (*) are mandatory for DCS submission.
Distance/speed plausibilityFlags implausible average speeds (too high/low) given distance and hours.Often caused by hours entered in minutes, or distance in km instead of NM.
📄 7. Annual Report

Compiles all the above tabs into the standardised DCS/FoC Annual Report format aligned with IMO MARPOL Annex VI for submission to the flag Administration or RO, plus tools to export/re-import all entered data as XML and to print the report.

ElementDescriptionCaution ⚠

⚠ General Cautions
  • This tool performs calculations based on the data you enter — it does not validate against ship certificates or bunker delivery notes automatically. Always cross-check source documents.
  • Always complete tabs in order: Ship Particulars → Voyage Data → Correction Factors → Fuel Consumption → CII Results → Verification → Annual Report, since later tabs depend on earlier inputs.
  • Data is held in the browser session only. Use Export XML regularly to avoid losing work on refresh or tab close.
  • Fuel emission factors (Cf) follow IMO MEPC guidance — only override with a custom fuel if a recognised, documented emission factor is available.
  • CII reduction factors are year-specific and update automatically based on the Reporting Year — always confirm this is correct first.
